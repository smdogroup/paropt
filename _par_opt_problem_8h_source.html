6 /*
7  This code is the virtual base class problem definition for the
8  parallel optimizer.
9 
10  To get this to work you must override the following virtual
11  functions:
12 
13  getVarsAndBounds(): This function returns the variables and bounds
14  for the problem. This is called once at the initialization. The
15  starting point is taken from the
16 
17  evalObjCon(): This function takes in the design variables and
18  returns an objective and constraint values. The function returns a
19  fail flag (e.g. fail = evalObjCon(x, &fobj, con);). Return fail != 0
20  if the function cannot be evaluated at the provided values of the
21  design variables.
22 
23  evalObjConGradient(): This function evaluates the objective and
24  constraint gradients at the current point. Again, the fail flag is
25  given as above. Note that the constraint gradients are returned as a
26  series of dense vectors.
27 
28  The class takes as input the communicator for the optimizer, the
29  number of local design variables on the given process, and the
30  number of constraints in the problem.
31 
32  input:
33  comm: the communicator
34  nvars: the number of local variables
35  ncon: the number of dense constraints
36  nwcon: the number of sparse constraints
37  nwblock: the block size of the Aw*D*Aw^{T} matrix
